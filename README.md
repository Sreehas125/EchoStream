# EchoStream

EchoStream is a Spring Boot platform for managing the full lifecycle of digital audio assets: ingestion, audit workflow, rights management, and royalty payouts.

## What This Project Does

EchoStream combines five operational areas into one application:

- Audio ingestion for music and podcast tracks
- Audit queue workflow with lifecycle transitions
- Rights and license management per track
- Royalty calculation using pluggable strategies
- Payment transaction tracking and edits

The app includes both:

- REST APIs under `/api/*`
- Thymeleaf-based web dashboards for operations workflows

## Lifecycle Model

Track lifecycle is controlled by the State pattern:

`UPLOADED -> VALIDATING -> PENDING_AUDIT -> RELEASED`

`REJECTED` can be reached from `VALIDATING` or `PENDING_AUDIT`.

Invalid transitions throw `InvalidStateTransitionException`.

## Quick Start

1. Create/update MySQL credentials to match `application.properties`.
2. Build the project:

```bash
mvn clean package
```

3. Run the application:

```bash
java -jar target/echostream-0.0.1-SNAPSHOT.jar
```

4. Open:

- `http://localhost:8081/` (redirects to ingestion dashboard)

## Initial Data Requirement

The application does not seed users automatically.

Before ingesting tracks or assigning audits, create at least:

- one `ContentCreator`
- one `QualityAuditor`

You can create these directly in MySQL using tables generated by JPA (`users`, `content_creators`, `quality_auditors`) after first startup.

## Web Dashboard Usage

### 1) Ingestion Dashboard

- Route: `/ingestion-dashboard`
- Upload local audio files and metadata
- Track-type file limits in controller logic:
  - MUSIC: 10 MB
  - PODCAST: 100 MB
- On success: track is created with initial lifecycle `UPLOADED`

### 2) Audit Queue

- Route: `/audit-queue`
- Shows inbound items (`UPLOADED`, `VALIDATING`) and pending audit queue (`PENDING_AUDIT`)
- Actions:
  - Queue for audit
  - Approve
  - Reject

### 3) Rights Management

- Route: `/rights-management`
- Shows tracks and latest license
- Actions:
  - Renew license (generates a new NON_EXCLUSIVE one-year license)
  - Revoke access (sets current date as `effectiveUntil` for existing licenses)

### 4) Royalty Report

- Route: `/royalty-report`
- Supports:
  - Fixed-rate payout calculation
  - Revenue-share payout calculation
- Creates a payment transaction on successful calculation

### 5) Transaction Edits

- Route: `/transaction-edits`
- Lets operators edit transaction currency and status
- Currency must be a 3-letter code

## REST API Reference

### Audio Ingestion

Base path: `/api/ingestion`

#### POST `/api/ingestion/tracks`

```json
{
  "trackType": "MUSIC",
  "creatorId": 1,
  "title": "Midnight Echo",
  "durationSeconds": 240,
  "filePath": "uploads/example.mp3",
  "albumName": "Neon Horizons",
  "genre": "Synthwave",
  "isrcCode": "USRC17607839"
}
```

Podcast variant uses:

- `podcastName`
- `episodeNumber`
- `hostName`

#### POST `/api/ingestion/legacy/music`

```json
{
  "creatorId": 1,
  "isrcCode": "USRC17607839",
  "metadata": {
    "songTitle": "Midnight Echo",
    "lengthSeconds": 240,
    "fileLocation": "uploads/example.mp3",
    "album": "Neon Horizons",
    "genreName": "Synthwave"
  }
}
```

#### POST `/api/ingestion/external/podcast`

```json
{
  "creatorId": 1,
  "isrcCode": "USRC17607840",
  "metadata": {
    "episodeTitle": "Episode 12: Rights in Streaming",
    "durationInSeconds": 1800,
    "storageUrl": "uploads/podcast-12.mp3",
    "showName": "Echo Talks",
    "episodeNo": 12,
    "host": "A. Rivera"
  }
}
```

### Rights Management

Base path: `/api/rights`

#### POST `/api/rights/tracks/{trackId}/licenses`

```json
{
  "licenseCode": "LIC-EXCLUSIVE-2026",
  "licenseType": "EXCLUSIVE",
  "territory": "US",
  "effectiveFrom": "2026-04-21",
  "effectiveUntil": "2027-04-21"
}
```

### Royalties

Base path: `/api/royalties`

#### POST `/api/royalties/calculate/fixed-rate`

```json
{
  "audioTrackId": 42,
  "streamCount": 50000,
  "fixedRatePerStream": 0.005,
  "grossRevenue": 0,
  "revenueSharePercent": 0
}
```

#### POST `/api/royalties/calculate/revenue-share`

```json
{
  "audioTrackId": 42,
  "streamCount": 50000,
  "grossRevenue": 10000,
  "revenueSharePercent": 0.12,
  "fixedRatePerStream": 0
}
```

#### POST `/api/royalties/transactions`

```json
{
  "audioTrackId": 42,
  "beneficiaryId": 1,
  "amount": 250.00,
  "currency": "USD"
}
```

### Audit Workflow

Base path: `/api/audit`

- `PUT /api/audit/tracks/{trackId}/assign/{auditorId}`
- `PUT /api/audit/tracks/{trackId}/start-validation`
- `PUT /api/audit/tracks/{trackId}/pending-audit`
- `PUT /api/audit/tracks/{trackId}/approve`
- `PUT /api/audit/tracks/{trackId}/reject`

## Error Handling

`ApiExceptionHandler` maps:

- `ResourceNotFoundException` -> HTTP 404
- `InvalidStateTransitionException` -> HTTP 400

Response format:

```json
{
  "error": "message text"
}
```

## Domain Design Patterns Used

- Adapter Pattern: normalizes external/legacy metadata into a unified model
- Factory Pattern: creates `MusicTrack` vs `PodcastTrack`
- State Pattern: enforces lifecycle transitions
- Strategy Pattern: switches royalty engines (fixed-rate vs revenue-share)

## Key Entities

- `User` (abstract)
  - `ContentCreator`
  - `QualityAuditor`
- `AudioTrack` (abstract)
  - `MusicTrack`
  - `PodcastTrack`
- `DigitalLicense`
- `PaymentTransaction`

- No authentication/authorization layer configured
- No seed/demo data bootstrap
